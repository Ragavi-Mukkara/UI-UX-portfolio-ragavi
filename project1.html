Case study

GITAM Student Portal UX Redesign

Redesigning the My‑GITAM student portal to enhance usability, clarity, and task completion for students, with a focus on core daily tasks like assignment submission.

Role

Junior UI/UX Designer

Focus

UX redesign, information architecture, visual design

Tools

Figma, FigJam, Notion

Prototype

View Figma prototype →

Problem

The existing student portal is functional but suffers from poor visual hierarchy, information overload, confusing navigation, and unclear primary actions, especially for daily tasks like assignment submission.

Students often struggle to quickly find and complete their tasks with confidence, leading to friction in everyday academic workflows.

Design goals

  • Focus on clarity and quick task completion.
  • Prioritize everyday needs such as assignments, quizzes, and attendance.
  • Create a more scannable, student‑friendly dashboard.
  • Reduce unnecessary cognitive load in key interaction flows.

User flow

The redesigned flow focuses on a real, frequent student task:

  • Login → Dashboard → Assignment Details → Upload Assignment → Submission Success

This flow ensures students can move from awareness to completion with minimal friction.

Screens & UX intentions

  • Login – Focuses users on the primary action without distractions.
  • Dashboard – Surfaces today’s academic priorities at a glance.
  • Assignment details – Provides clarity on requirements before submission.
  • Upload – Minimizes errors with clear file upload instructions and CTAs.
  • Success – Reassures users with confirmation and submission details.

Key UX decisions

Simplified login

  • Removed clutter and unnecessary navigation elements.
  • Centered core content to improve focus and reduce confusion.

Prioritized dashboard

  • Introduced “Today’s Tasks” as a central dashboard element.
  • Made assignments and quizzes easily visible and scannable.

Task‑focused experience

  • Students can see tasks with due times and clear actions like “Submit” or “Start”.
  • Icons and subtle colour cues improve recognition and decision making.

End‑to‑end assignment flow

  • Provides context before action via clear assignment details.
  • Guides uploads with file hints and clear CTAs.
  • Confirms success to reduce anxiety after submission.

Key screens

Before vs after

Aspect Before After
Navigation Cluttered with multiple areas competing for attention. Centered and task‑first.
Task visibility Hidden or buried within the portal. Prominent on the dashboard.
Upload clarity Poor guidance and affordances. Clear upload guidelines and CTAs.
User reassurance Minimal confirmation and feedback. Dedicated confirmation screen with details.

Impact & takeaways

This redesign improves task completion visibility, reduces confusion about where to go and what to do, and creates a more modern, student‑friendly interface.

It strengthened my understanding of user flows, action prioritization, visual hierarchy, and interface affordances.
